Architecture · Section 05

Defence Layers (Spec A)

Eight independent, layered defences spanning the web middleware, LangGraph graph node, and compliance-internal verifier — each with a single clear responsibility and a dedicated audit trail. Designed to be testable in isolation and unconditional on critical limits.

1

The 8 Layers

Where each defence lives and what it stops

B1 · Web Middleware
Rate + Length Caps
20 msg/min/session · message ≤2 000 chars · justification ≤500 chars · control chars rejected → 429/413
B2 · abuseGuard
Coherence Gate
Min 10 chars · non-stopword ratio ≥30% · must reference known expense category or recognised reason phrase
B3 · Chat Router
Injection Firewall
Strips "ignore instructions", [INST], system:, long base64 blobs. Wraps all user text in <user_input> fence
B4 · abuseGuard
Cross-Check LLM
One cheap LLM call (temp=0, ~150 tokens): does the justification plausibly explain this receipt? → inconsistency → requiresReview
B5 · Compliance
Hard Monetary Caps
SGD 5 000/receipt · SGD 10 000/claim · SGD 20 000/employee/month. Deterministic — unconditional, no LLM override
B6 · Compliance
Self-Critique Verifier
2nd LLM call, different model family, temperature 0.0, different system prompt. Disagrees → finalVerdict = requiresReview
B7 · Ubiquitous
Audit Trail
Every boundary trip writes one audit_log row with actor, action, details, timestamp. Available on /audit and mmga logs grep
B8 · Web Middleware
Per-User Quotas
20 submissions/user/day · 5 retries/user/hr after fail verdict. Stored in user_quota_usage table → 429 on trip

Violation Severity & Justification Rules (W2)

Which violations can be flipped by a user justification, and which are unconditional

Violation type Severity class Justification can flip? Flip target
Amount > cap (≤150% of cap) soft ✅ Yes — if coherent justification provided requiresManagerApproval
Missing preferred vendor soft ✅ Yes requiresManagerApproval
Expense outside working hours soft ✅ Yes requiresManagerApproval
Amount 150%–B5 cap soft-plus ✅ Yes — with stronger justification requiresDirectorApproval
Alcohol / entertainment outside allowlist hard ❌ Never Verdict = fail
Non-claimable category (personal, gift) hard ❌ Never Verdict = fail
Amount > SGD 5 000 / SGD 10 000 / SGD 20 000 (B5) hard cap ❌ Never — deterministic, no LLM Auto requiresDirectorApproval
Exact duplicate receipt hard ❌ Never Fraud flag → escalate
